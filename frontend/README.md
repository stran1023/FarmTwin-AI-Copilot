# Frontend

Next.js 16 (App Router, TypeScript, Tailwind CSS v4, shadcn/ui) client for
the FarmTwin AI Copilot. Originally generated by v0.app against an
in-memory mock; now wired to the real FastAPI backend in `../backend`. See
`../docs/frontend-architecture.md` for the full architecture writeup and
`../docs/ui-build-plan.md` for the original screen-by-screen design intent.

## Setup

```bash
npm install
cp .env.local.example .env.local
npm run dev
```

Backend must be running for any page to show real data:

```bash
cd ../backend && source venv/Scripts/activate  # or venv/bin/activate
uvicorn app.main:app --reload
```

### Environment

`.env.local` (gitignored):

```
NEXT_PUBLIC_API_URL=http://localhost:8000
```

All API calls in `lib/api.ts` point at this URL.

## Pages

Per `../docs/ui-build-plan.md`, with one deviation noted below:

1. `/` and `/assets/[id]` — one shell (`components/SplitFarmView.tsx`): an
   isometric, pan/zoom digital twin map on the left (`GET /assets`), and a
   right panel that swaps in place between the farm dashboard
   (`GET /dashboard/summary`) and an asset's detail
   (`GET /assets/{id}`, `GET /assets/{id}/recommendations`) — click a
   marker or a dashboard row to select an asset, no page navigation or
   remount. Approve/Reject on a recommendation writes to Snowflake via
   `POST /recommendations/{id}/approve|reject`.
2. `/copilot` — free-form Q&A grounded in live farm state
   (`POST /copilot/ask`). **Deviation from the build plan**: this is a
   dedicated route, not a persistent panel mounted in the root layout, so
   the conversation resets on navigation. Flagged in
   `../docs/frontend-architecture.md`, not yet changed.
3. `/briefing` — today's approved/rejected recommendations plus a
   Cortex-Agent-generated summary (`GET /briefing/today`). Cortex Agent
   calls are real and can take 40-150s — this page's loading state is not
   a hang.

## Architecture notes

- `lib/api.ts` is a **mapping layer**, not a passthrough: it fetches the
  real backend (shapes mirroring `backend/app/models/schemas.py`) and
  converts every response onto this frontend's own `lib/types.ts`
  contract (e.g. `confidence_pct` 0-100 → `confidence` 0-1, growth-stage
  string enum → 0-4 index, `pending_approval` → `pending`). See
  `../docs/frontend-architecture.md` for the full mapping rationale.
- `lib/dataCache.ts` + `lib/useApiData.ts`: a hand-rolled fetch cache
  (TTL + in-flight dedup + cross-component `invalidate()`), no new
  dependency. Multiple components reading the same cache key (e.g.
  `"dashboard-summary"`) share one fetch and re-render together when any
  of them writes and invalidates it.
- No websockets — writes (approve/reject, asking the copilot) invalidate
  the relevant cache key and refetch; every mounted consumer picks up the
  change automatically.

## Known issues

- The AI Copilot is not currently a persistent cross-navigation panel
  (see `/copilot` above).
